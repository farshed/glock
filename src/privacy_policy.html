Glock — Privacy Policy

Last updated: 17 June 2026

Glock is a browser extension that displays a lines-of-code badge on GitHub repository pages. This policy explains what data the extension and its companion API handle, and how.

What we collect

  • Repository identifiers. When you view a GitHub repository, the owner/repo name of that repository is sent to the Glock API so it can count its lines of code.
  • GitHub Personal Access Token (optional). If you choose to add a token to count private repositories, that token is stored locally in your browser and sent to the Glock API only to clone the repositories you view.

We do not collect names, email addresses, browsing history, analytics, or any other personal information.

How your data is used

  • Repository identifiers and (when provided) your token are used solely to clone the requested repository, count its lines of code, and return that number to the extension.
  • Cloned repositories are written to a temporary directory and deleted immediately after counting. They are not retained.
  • Your token is never logged. It is held only for the duration of a clone request and is not written to disk by the API.

Storage

Your Personal Access Token and cached line counts are stored locally in your browser using the extension storage API. They never leave your device except to make the requests described above, and you can remove them at any time by clearing them in the extension options or removing the extension.

Sharing

We do not sell, rent, or share your data with any third party. Data is sent only to the Glock API instance that you configure, and to GitHub when cloning a repository.

Data retention

The API does not persist repository contents, identifiers, or tokens beyond the lifetime of an individual request. Locally stored data remains in your browser until you delete it.

Contact

Questions about this policy can be sent to mail@farshed.me.

Glock is open source. This policy applies to the official extension and API.
